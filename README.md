# books-download-extension

Расширение Chrome (Manifest V3) — учебный проект, исследующий механизмы аутентификации, обработку кросс-доменных запросов и сборку EPUB-файлов на примере тайваньской платформы электронных книг.

Проект демонстрирует несколько характерных для MV3 паттернов: инъекцию fetch-запросов через `chrome.scripting.executeScript` для обхода ограничений на cookie в service worker, очередь загрузок в памяти с учётом keep-alive, загрузку файлов через base64 `data:` URL как замену недоступному в MV3 `URL.createObjectURL`, а также полную сборку EPUB/KEPUB прямо в браузере без серверной части.

## Установка

### Требования

- [Node.js](https://nodejs.org/) >= 18
- Google Chrome

### Сборка

```bash
npm install
npm run build
```

В результате создаётся папка `dist/` с готовым расширением.

### Загрузка в Chrome

1. Откройте Chrome и перейдите на `chrome://extensions`
2. Включите **Режим разработчика** (переключатель в правом верхнем углу)
3. Нажмите **Загрузить распакованное расширение**
4. Выберите папку `dist/`

Иконка **books-download** появится на панели инструментов Chrome.

## Использование

1. Войдите в аккаунт [books.com.tw](https://www.books.com.tw) в Chrome
2. Откройте [полку электронных книг](https://viewer-ebook.books.com.tw/viewer/index.html?readlist=all&MemberLogout=true) — расширению требуется, чтобы эта страница была открыта
3. Нажмите на иконку расширения **books-download**
4. Отобразится список купленных книг
5. Выберите формат вывода с помощью переключателя вверху:
   - **EPUB** — стандартный формат, поддерживается большинством ридеров и приложений
   - **Kobo KEPUB** — вариант EPUB от Kobo, рекомендуется для устройств Kobo (лучшая типографика и рендеринг)
6. Нажмите **[DL]** рядом с книгой для начала загрузки
7. Файл сохранится в папку загрузок Chrome по умолчанию

## Настройки

Расширение имеет панель **Settings** (доступна через меню в правом верхнем углу):

| Параметр                  | По умолчанию | Описание                                                                 |
| ------------------------- | ------------ | ------------------------------------------------------------------------ |
| Задержка запросов файлов  | 1000 мс      | Пауза между запросами отдельных файлов при скачивании книги              |

**Не устанавливайте слишком маленькую задержку.** Каждый EPUB состоит из множества файлов, которые скачиваются по одному. Слишком частые запросы могут вызвать блокировку аккаунта. Значение по умолчанию 1000 мс является безопасным. Значения ниже 600 мс не рекомендуются, ниже 300 мс — с высокой вероятностью вызовут срабатывание защиты от ботов.

## Модуль криптографии

Реализация расшифровки (`src/crypto/crypto.js`) не включена в репозиторий — опубликованный файл является заглушкой, выбрасывающей `Not implemented`.

Для сборки рабочего расширения необходимо предоставить собственный файл `src/crypto/crypto.real.js`, экспортирующий:

| Экспорт       | Сигнатура                                         | Описание                                                                   |
|---------------|---------------------------------------------------|----------------------------------------------------------------------------|
| `generateKey` | `async (url, downloadToken) → Uint8Array`         | Формирует 32-байтовый XOR-ключ из URL файла и `download_token`             |
| `decodeXor`   | `(key, encryptedBytes) → Uint8Array`              | Применяет ключ циклически; удаляет ведущий UTF-8 BOM при наличии           |
| `imgChecksum` | `() → string`                                     | Возвращает 16-символьную hex-строку, используемую при регистрации устройства |

Файл добавлен в `.gitignore`. При его наличии Vite автоматически использует его вместо заглушки во время сборки — никаких других изменений не требуется.

## Вопросы и ответы

### books.com.tw продаёт книги, не предоставляя покупателям доступ к самим файлам. Этично ли публиковать подобный инструмент?

Подобно многочисленным инструментам DeDRM для Amazon Kindle и других платформ, этот инструмент предназначен прежде всего для создания резервных копий уже оплаченных книг и демонстрации технической возможности.

Веб-читалка books.com.tw работает медленно, общий опыт использования оставляет желать лучшего, и — в отличие от Kobo, который хотя бы предоставляет доступ к EPUB-файлам — books.com.tw не позволяет читать купленные книги на устройстве или в приложении по выбору пользователя. Купив бумажную книгу, вы ставите её на полку; купив электронную, вы должны иметь возможность открыть её в собственном ридере.

Инструмент скачивает только книги, привязанные к вашему собственному аккаунту. Он не обходит оплату.
